# UI Specification

The UI must prioritize shop operations over visual complexity.

## Principles

- Fast
- Simple
- Touch friendly
- Mobile friendly
- Minimal training required
- Important actions reachable quickly
- Regional language support
- Clear success/error feedback

## Dashboard

Possible sections:

### Daily Operations

Sell
Return
Inventory
Customers

### Management

Purchase
Vendor
Expenses
Employees
Dues

### Intelligence

Reports
Analytics
AI Advisor

### Relationships

WhatsApp
Birthdays
VIP Customers
Inactive Customers

## Today's Attention

Dashboard should eventually surface:

- Low stock
- Payments due
- Customer birthdays
- Important customer follow-ups
- Dead stock
- Pending purchases
- Business performance alerts

## Screen Documentation Standard

Every screen specification should contain:

Purpose
Roles
Fields
Buttons
Actions
Validation
Business Rules
API Calls
Success Flow
Failure Flow
Edge Cases
Responsive Behaviour
